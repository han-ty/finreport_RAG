  项目全景分析：上市公司财报"智能问数"助手

  这是什么项目？

  这是 2026年泰迪杯数据挖掘挑战赛 B题 的完整解决方案 —— 一个基于大语言模型（LLM）的 上市公司财报智能问答系统，实现了"自然语言查财报"的完整技术链路。

  核心技术架构

  用户输入自然语言问题
      ↓
  [意图分析]  识别查询类型（查询/排名/对比/趋势/归因）
      ↓
  [SQL生成]   LLM将自然语言 → SQLite SQL
      ↓
  [查询执行]   安全执行SQL，返回结构化数据
      ↓
  [可视化]     自动选择合适的图表类型（折线/柱状/饼图）
      ↓
  [RAG检索]    从研报知识库检索相关上下文
      ↓
  [回答生成]   LLM综合SQL结果 + 知识库 → 专业分析报告
      ↓
  [归因分析]   追溯每条结论的证据来源

  技术栈

  第一阶段：跑起来，建立感性认识（1-2天）

  1. 配置环境并运行
    - 创建 conda 环境，安装依赖
    - 修改 config.json 中的 project_root 和 API Key
    - 依次运行 task1/run_task1.py → task2/run_task2.py → task3/run_task3.py
    - 启动前后端：python -m backend.api.server + cd frontend && npm run dev
    - 目标：看到整个系统跑通，理解输入输出是什么
  2. 理解数据流
    - 看 项目目录结构.md 了解文件组织
    - 打开 data/financial.db（用 DB Browser for SQLite）看5张表的结构和内容
    - 打开 results/result_2.xlsx 和 results/result_3.xlsx 看最终输出

  第三阶段：简历可写的"技术亮点"（理解后自行发挥）

  掌握了以上模块后，你的简历可以写这些硬核技术点：

  1. RAG（检索增强生成）系统

  ▎ "设计并实现了基于 BGE 嵌入模型 + 向量检索 + LLM 归因分析的 RAG  Pipeline，将非结构化研报知识库与结构化数据库查询结果融合，回答准确率提升 20%+"

  你要能说清：文档分块策略（chunk_size/chunk_overlap）、向量检索原理、混合检索（向量+关键词）、重排序（Reranker）的概念。

  2. NL2SQL（自然语言转SQL）

  ▎ "基于 LLM 的 Prompt Engineering 实现自然语言到 SQL 的自动转换，包含意图分析、Schema 描述注入、Few-shot 示例、SQL 安全校验等环节"

  你要能说清：为什么要在 Prompt 里放 Schema 描述？为什么要做意图分析？SQL 安全检查做了什么？

  3. Agent 架构设计

  ▎ "设计多步骤 Agent 工作流：意图分析 → 任务规划 → SQL 生成 → 查询执行 → 可视化 → 多源结果融合 → 归因分析，支持多轮对话和意图澄清"

  你要能说清：Agent 的每一步在做什么？为什么需要意图澄清？多意图规划解决什么问题？

  4. LLM 工程化实践

  ▎ "实现多 LLM Provider 负载均衡、自动重试、JSON Mode 结构化输出、SSE 流式响应，降低 API 成本 30%"

  你要能说清：JSON Mode 是什么？SSE vs WebSocket 的区别？为什么要加权随机选择 Provider？

  5. PDF 结构化数据提取

  ▎ "使用 pdfplumber 实现上市公司 PDF 财报自动解析，支持上交所/深交所两种命名格式，提取四大财务报表（核心指标/资产负债/利润/现金流）的结构化数据"

  简历写法示例

  ▎ 项目：上市公司财报智能问数助手 | Python, FastAPI, SQLite, React, LLM
  ▎ 泰迪杯数据挖掘赛 B 题完整解决方案
  ▎ - 设计 RAG Pipeline：集成 BGE 嵌入向量检索 + LLM 生成，融合结构化财务数据和研报知识库，实现带来源引用的分析报告
  ▎ - 实现 NL2SQL 引擎：通过 Prompt Engineering（Schema 注入 + Few-shot + 意图分析）将自然语言自动转换为 SQL，支持单表查询/多表 JOIN/聚合排名/同比环比
  ▎ - 构建 多步骤 Agent 架构：意图分析→任务规划→SQL生成→查询执行→可视化→归因分析，支持多轮对话和意图澄清
  ▎ - 全栈开发：FastAPI + SSE 流式后端，React + TypeScript + Ant Design 前端
